# Accordo IDE

AI-native development environment built on VS Code.

- MCP tools for every editor action
- Agent controls editor, terminal, and browser
- Comments as the human-agent task channel

<!-- notes -->

Welcome to the Accordo IDE demo. This slide introduces the project. Accordo gives AI agents direct, structured access to the VS Code editor through the Model Context Protocol.

---

# Architecture

Three-layer system.

- **Hub** — standalone MCP server, editor-agnostic
- **Bridge** — VS Code extension, connects editor to Hub
- **Editor** — 16 tools for file, terminal, and layout control

<!-- notes -->

This slide covers the high-level architecture. Hub is a Node.js server that speaks MCP. Bridge is a VS Code extension that exposes editor capabilities. Editor tools cover opening files, running commands, and controlling layout.

---

# Comment System

Human and agent collaborate through comment threads.

- Place comments on code lines, slides, diagrams, or markdown
- Agent creates, replies, resolves threads as MCP tools
- Comments visible in the agent system prompt

<!-- notes -->

The comment system is the primary channel for human-agent communication. Rather than chat, the human places a comment directly on the artifact they care about. The agent sees open threads in every prompt.

---

# Presentation Modality

What you are looking at right now.

- Open any Slidev `.md` deck from a MCP tool call
- Navigate slides programmatically (`goto`, `next`, `prev`)
- Generate narration text per slide for TTS
- Annotate slides with the Comment SDK

<!-- notes -->

This is a self-referential slide. The accordo-slidev extension is what opened this deck. The agent can navigate to any slide, read its narration, and drop a comment anywhere on the canvas.

---

# Narration Example

Speaker notes become TTS-ready plain text.

```markdown
<!-- notes -->
Speak this text aloud to the audience.
Bold and *italic* are stripped automatically.
```

No markdown syntax in the output — ready for voice synthesis.

<!-- notes -->

Narration is generated by stripping all markdown syntax from speaker notes, or from headings and bullets when no notes are present. The output is plain text suitable for direct TTS consumption by the accordo-voice extension in phase four.

---

# What's Next

Roadmap ahead.

- **Phase 4:** Voice (`accordo-voice`) — narration playback
- **Phase 5:** Diagrams (`accordo-tldraw`) — canvas annotations
- **Phase 6:** Browser (`accordo-browser`) — Playwright integration

<!-- notes -->

The presentation modality is phase three. Phase four adds voice playback by connecting the narration output to a TTS engine. Phase five adds tldraw diagram control. Phase six brings in Playwright so the agent can control a browser tab as naturally as it controls the editor.
