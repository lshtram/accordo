## FAIL — must fix before Phase E

### [S-04-F1] 🔴 Missing test: edge key migration on rename
**File:** `packages/diagram/src/__tests__/reconciler.test.ts`  
**Context:** The entire new code path in `reconciler.ts` lines 119–136 (the `updatedEdges` loop) has **zero direct test coverage**. Tests RC-19..RC-23 only assert that renamed node keys appear in `layout.nodes` — none assert that edge keys are also migrated.

**Required fix:** Add at least one test (ideally two) that:
1. Starts with a layout containing `edges: { "auth->api:0": { ... } }`
2. Renames `auth` → `new_auth`
3. Asserts `layout.edges["new_auth->api:0"]` exists with the original routing data intact
4. Asserts `layout.edges["auth->api:0"]` no longer exists

A second test should cover a rename affecting the `to` node (e.g. `api` → `gateway`), producing `auth->gateway:0`.

### [S-04-F2] 🔴 Import style violation

**Current code:**
```typescript
const updatedEdges: Record<string, import("../types.js").EdgeLayout> = {};
```

### [S-05-F1] 🔴 Type signature missing BT / RL
**File:** `packages/diagram/src/reconciler/placement.ts`, line 82  
**Current code:**
```typescript
direction?: "TD" | "LR";
```
**Problem:** The implementation handles all 4 directions, but the public `options` type only permits `"TD"` and `"LR"`. Any caller passing `direction: "BT"` or `direction: "RL"` via options receives a TypeScript type error — the public API contract contradicts the implementation.

**Required fix:**
```typescript
direction?: "TD" | "LR" | "BT" | "RL";
```

### [S-05-F2] 🔴 JSDoc outdated

```typescript
 * @param options.direction   Flow direction — "TD" (default) or "LR"
```

### [S-05-F3] 🔴 No tests for BT or RL direction
**File:** `packages/diagram/src/__tests__/placement.test.ts`  
**Problem:** `placement.test.ts` has TD tests (PL-03) and LR tests (PL-04, PL-18) but **zero tests for `direction: "BT"` or `direction: "RL"`**. Both new switch branches in the dagre-path and the fallback heuristic are completely untested.
